## Коллекции используются для хранения набора данных.
В java есть несколько различный вариантов коллекций, они различаются:
1. безопасностью использования в многопоточных средах
2. возможностью хранить дублирующиеся элементы
3. возможностью хранить элементы в отсортированном виде
4. скоростью добавления, удаления, извлечения элементов из начала / конца / середины коллекции
5. возможностью хранить элементы в том порядке, в котором они были добавлены и т.д.

Все коллекции из данного списка не являются потокобезопасными.
В классах, экземпляры которых необходимо хранить в коллекциях почти всегда переопределяются equals и hashCode

## Особенности LinkedList:
1. быстрая вставка в середину и удаление из середины списка, во всех остальных случаях необходимо выбирать ArrayList
2. элементы хранятся в порядке добавления
3. можно добавить null
4. допускает дублирование элементов
5. используется довольно редко, обычно предпочтение отдается ArrayList

## Особенности ArrayList:
1. реализован на основе массива
2. элементы хранятся в порядке добавления
3. можно добавить null
4. допускает дублирование элементов
5. используется очень часто

### Особенности Set:
1. используется для хранения уникальных элементов
2. обязательно должны быть переопределены методы equals и hashcode

## Особенности HashSet:
1. основан на hash таблице (использует hash code для размещения элементов)
2. порядок хранения элементов может отличаться от порядка добавления
3. можно хранить null (всегда будет на 1 месте)

## Особенности LinkedHashSet:
1. более медленный, чем HashSet
2. основан на hash таблице (использует hash code для размещения элементов)
3. порядок хранения элементов тот, в котором они добавлялись
4. можно хранить null 

## Особенности TreeSet:
1. нельзя добавить null
2. элементы хранятся в отсортированном порядке
3. основан на алгоритме красно-черного бинарного дерева
4. для добавления элементов в TreeSet необходимо:
    1) чтобы класс, экземпляры которого будут храниться в коллекции, реализовывал интерфейс Comparable и его метод compareTo
    2) передать в конструктор TreeSet объект Comparator (данный способ имеет больше возможностей), который описывает правила сортировки объектов
